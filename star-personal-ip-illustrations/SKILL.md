# Star Personal IP Illustrations

Generate concise illustrations with Star's recurring personal IP. The point is not a generic engineer mascot: show a petite independent creator turning scattered complexity into a clear, usable result. Keep one shared identity while choosing both a character mode (`main-ip` or `chibi-sticker`) and a rendering style (`clean-line` or `crayon-blue`).

## Read first

- Read `references/personal-ip.md` for the non-negotiable character identity and visual grammar.
- Read `references/prompt-template.md` before generating or editing an image.
- Read `references/style-dna.md` to choose a style. Then read exactly one matching profile: `references/styles/clean-line.md` or `references/styles/crayon-blue.md`.
- Read `references/composition-patterns.md` when the source needs a new explanatory layout.
- Read `references/qa-checklist.md` after generation.
- Use `assets/star-main-ip-reference.png` for the regular full-body IP. Use `assets/star-chibi-ip-reference.png` for small Q-version accents and sticker-like scenes.
- Read `references/example-set.md` and use only the example that matches the current need. Do not use the legacy Xiaohei examples.

## Workflow

1. Extract one visual anchor from the source: a decision, a broken link, a delivery path, or a result.
2. Select a character mode. Use `main-ip` for covers, central scenes, work narratives, and substantial actions. Use `chibi-sticker` for page corners, reactions, short transitions, badges, and light decorative moments.
3. Select a rendering style. Honor an explicit user choice; otherwise use `clean-line` for formal review-deck and explanatory work, and `crayon-blue` for personal, editorial, or playful work. When uncertain, default to `clean-line`.
4. Design one image around that anchor. Keep the character responsible for the action, including in chibi mode.
5. Choose one visual grammar from `references/personal-ip.md` that fits the story.
6. Generate one image at a time, then check the shared identity, selected character mode, and selected rendering style.

## Output rules

- Default to pure white 16:9 images with generous empty space and no title text. A user may explicitly request another aspect ratio, such as a square character portrait.
- Keep Chinese labels optional; use at most four, each under six Chinese characters.
- Do not repeat a motif merely because it appears in a previous image. Pick the smallest metaphor that explains the current idea.
- Use the selected style profile consistently; never mix clean black line art and cobalt crayon rendering accidentally.
- Do not use realism, office stock art, a generic anime heroine, or a child mascot.
- Always keep the vivid red hair bow on the character's left side, the pink square smartwatch on her left wrist, and the small gold pendant. Do not mirror or omit them.
- Keep the selected mode's canonical summer outfit and proportions unless the user explicitly asks for a different outfit or season.
- Preserve user-provided facial and hairstyle details when a current reference photo is supplied; never invent a different identity.
