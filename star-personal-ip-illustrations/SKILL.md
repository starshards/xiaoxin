# Star Personal IP Illustrations

Generate concise 16:9 explanatory illustrations with Star's recurring personal IP. The point is not a generic engineer mascot: show a petite independent creator turning scattered complexity into a clear, usable result.

## Read first

- Read `references/personal-ip.md` for the non-negotiable character identity and visual grammar.
- Read `references/prompt-template.md` before generating or editing an image.
- Read `references/style-dna.md` for palette and overall tone. Read `references/composition-patterns.md` when the source needs a new explanatory layout.
- Read `references/qa-checklist.md` after generation.
- Use `assets/star-summer-ip-reference.png` as the primary character reference whenever the image tool accepts local reference paths.
- Read `references/example-set.md` and use only the example that matches the current need. Do not use the legacy Xiaohei examples.

## Workflow

1. Extract one visual anchor from the source: a decision, a broken link, a delivery path, or a result.
2. Design one image around that anchor. Keep the Q-version character as the agent of the action, never a decorative sticker.
3. Choose one visual grammar from `references/personal-ip.md` that fits the story. Card → thin gold line → gold sparkle is one option for connection and verified delivery, not the default for every image.
4. Generate one image at a time, then check the required identity, compact proportions, outfit, and visual clarity.

## Output rules

- Default to pure white 16:9 images with generous empty space and no title text.
- Keep Chinese labels optional; use at most four, each under six Chinese characters.
- Do not repeat a motif merely because it appears in a previous image. Pick the smallest metaphor that explains the current idea.
- Use flat Q-version cartoon line art, not realism, office stock art, a generic anime heroine, or a child mascot.
- Keep the summer IP's clothing and petite adult proportions unless the user explicitly asks for a different outfit or season.
- Preserve user-provided facial and hairstyle details when a current reference photo is supplied; never invent a different identity.
